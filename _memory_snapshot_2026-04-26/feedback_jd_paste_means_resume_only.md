## The rule

**When Jeff pastes a JD, my default response is the resume file path. One line. That's it.**

The JD paste is not a request for analysis, fit assessment, gap flagging, or honest reads. It is the GO signal. Jeff has already vetted the URL on his end (per the locked URL-first protocol) and pasted the JD because he is at the application form and needs the resume version IMMEDIATELY so it parses into the form fields.

## Why this matters

Jeff at the form is in flow state. Every extra paragraph from me is friction. The resume file path is the answer. Anything more is delay.

## What "resume version only" means

```
[Resume file path]
```

That's it. Maybe one sentence on which variant rule applied if it's non-obvious (e.g., "VP variant — founding-tier seat" if there could be ambiguity). Otherwise just the path.

## When analysis IS welcome

If Jeff explicitly asks for analysis ("what about this stands out," "honest take," "advise me," "what do you think"), then deliver analysis. The default of resume-only ONLY applies when he pastes a JD without an explicit ask.

## When the resume is non-trivial

If the JD reveals a leadership-scale gate trigger (Miro-tier, post-acq large-co parent, "show me you've led big orgs") or a different gate (cyber → AT&T re-add, Japanese-HQ → Time T.I. re-add), I generate the tailored variant in one shot and surface it. Don't pause to ask.

If the variant choice is genuinely ambiguous (e.g., role title is sub-CRO but the scope is CRO-equivalent), give the answer + one-line reasoning, not a multi-paragraph explanation.

## Locked workflow

1. Jeff pastes JD → I respond with resume file path. ONE LINE.
2. Jeff submits.
3. Jeff says "logged" or "next" → I log + surface next URL pair.

Anything else is friction.
