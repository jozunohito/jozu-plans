The Plan

90 Days to a Working Sales Engine

Phase 1: Days 1-30
Take Over the Queue, Close the First Deal
✓ Own inbound immediately. Take over the inbound pipeline on day one. Fast, thoughtful qualification and follow-up. This is the current bottleneck. Fix it first. Every lead gets a response that demonstrates product knowledge and buyer understanding.
✓ Learn the product and demo independently. Get credible enough to run product demos solo. Understand APIs, SDKs, context graphs, and the Graphiti architecture well enough to hold a technical conversation with AI engineers without needing backup.
✓ Close a $15K+ deal. Run the full cycle: discovery, demo, security review, pricing proposal, contract negotiation, close. Prove the motion works in my hands, not just the founder's.
✓ Define the founder collaboration model. Work with Daniel to establish when he joins deals (technical deep-dives, executive alignment on large accounts) and when I run independently. Make the dynamic productive, not chaotic.
Phase 2: Days 31-60
Build and Run Systematized Outbound
✓ Launch outbound sequences. Define target accounts from the ICP. Build sequences with clear targeting, messaging, and cadence. Test and iterate on reply rates. Turn outbound from ad hoc into a system.
✓ Sales enablement buildout. Take the rough-form collateral and make it repeatable: demo scripts, competitive positioning docs, reference architectures, security questionnaire templates, pricing proposals. Every asset built to close, not to impress.
✓ Pipeline operations. Build reporting that shows what is working. Forecast accurately. Create the CRM hygiene and pipeline management discipline that lets us know where revenue is coming from and where it is stuck.
✓ Enterprise deal execution. Manage active enterprise opportunities through security reviews, procurement, and contract negotiation. Build the playbook for navigating enterprise compliance motions.
Phase 3: Days 61-90
Close a $50K+ Deal, Build the Foundation to Scale
✓ $50K+ deal closed. At least one significant enterprise deal through the full cycle. This validates the process, the pricing, and the ability to navigate complex procurement at this price point.
✓ Forecastable pipeline. A pipeline I can forecast against with confidence. Clear stages, clear next steps, clear decision timelines. No deals in the forecast based on hope.
✓ Hiring blueprint. The operational foundation that makes hiring the next person possible: documented process, enablement materials, defined role, comp plan, ramp plan. When we hire, they onboard in days.
✓ Open source to enterprise conversion path mapped. Define how Graphiti's 24K GitHub stars translate into enterprise pipeline. Build the developer-to-enterprise conversion motion: identify product-qualified leads from usage data, map the path from individual engineer to VP Engineering budget.
